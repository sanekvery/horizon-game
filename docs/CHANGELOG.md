## Фаза 1.2: Привязка игр к фасилитатору (завершено)

**Дата:** 2024-03-20

### Что сделано
- ✅ Создание игры через личный кабинет с выбором количества игроков
- ✅ Привязка сессии к facilitator_id в PostgreSQL
- ✅ Генерация уникальных кодов сессий (6 символов)
- ✅ Список игр фасилитатора с фильтрацией по статусу
- ✅ API для управления сессиями (`/api/sessions`)
- ✅ Проверка лимитов подписки (FREE: до 4 игроков)
- ✅ JWT авторизация в админке (без ввода пароля для залогиненных)
- ✅ Количество игроков сохраняется в сессии и используется везде
- ✅ Страница настройки: playerCount read-only, выбор сложности/режима
- ✅ Session code передаётся между страницами (`/admin` → `/setup` → `/qr`)
- ✅ Ссылка "Мои игры" в админке для возврата в кабинет

### Что проверить
1. Войти в `/facilitator`
2. Нажать "Создать игру" → выбрать количество игроков (4 для FREE)
3. Увидеть игру в списке со статусом "Подготовка"
4. Нажать "Управление" → попасть в админку автоматически (без пароля)
5. Нажать "Настройка игры" → увидеть количество игроков (read-only)
6. Выбрать сложность и режим распределения
7. Нажать "Начать распределение" → перейти к QR-кодам
8. Вернуться в личный кабинет по ссылке "Мои игры"

### Архитектура
```
/facilitator          → Создание игры (playerCount → PostgreSQL)
     ↓
/admin?session=CODE   → Админка (JWT авторизация)
     ↓
/setup?session=CODE   → Настройка (playerCount read-only, выбор сложности)
     ↓
/qr?session=CODE      → QR-коды для ролей
```

### Ограничения текущей версии
- ⚠️ Все игры используют общее игровое состояние (SQLite)
- ⚠️ Мультисессионность будет в Фазе 2
- ⚠️ Удалить можно только игры в статусе "Подготовка" или "Завершена"
