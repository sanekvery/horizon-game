- `components/map/index.ts` — НОВЫЙ: экспорт компонентов

### Socket-события

| Событие | Описание |
|---------|----------|
| `map:resource-contributed` | Ресурсы внесены в зону |
| `map:zone-upgraded` | Зона улучшена |
| `map:event-started` | Событие активировано |
| `map:event-ended` | Событие завершено |

### Что проверить
1. Открыть карту `/map?session=CODE`
2. В админке дать ресурсы игроку
3. Игрок вносит ресурсы в зону → видна анимация потока
4. Ресурсы отображаются на индикаторе зоны
5. Админ делает апгрейд зоны → красивая анимация
6. Активировать событие → зона подсвечивается
7. Открыть `/map?session=CODE&projector=true` → режим проектора

### Что НЕ входит (отложено на следующие этапы)
- Звуковые эффекты
- Погодные эффекты
- Персонализированные аватары игроков на карте

## Фаза 2: Мультисессионность (завершено)

**Дата:** 2026-03-21

### Что сделано
- ✅ Игровое состояние хранится в PostgreSQL (`game_sessions.state`)
- ✅ WebSocket rooms изолированы по sessionCode (`session:{code}`)
- ✅ Per-session таймеры (Map<sessionCode, NodeJS.Timeout>)
- ✅ Все GameService методы переведены на async + sessionCode
- ✅ API `/api/init-session` для инициализации состояния игры
- ✅ Все клиентские страницы передают sessionCode между собой
- ✅ QR-коды содержат параметр `?session=CODE`

### Затронутые файлы

**Server:**
- `prisma-game-state-repository.ts` — НОВЫЙ: репозиторий для PostgreSQL
- `game-service.ts` — все 50+ методов теперь async с sessionCode
- `socket-handler.ts` — session rooms, per-session timers, broadcastState(sessionCode)
- `api-routes.ts` — все endpoints принимают ?session=CODE
- `index.ts` — использует PrismaGameStateRepository

**Client:**
- `useSocket.ts` — auto-join session, isSessionJoined
- `useGameState.ts` — sessionCode awareness
- `AdminDashboard.tsx` — инициализация сессии через API
- `GameSetup.tsx` — sessionCode из URL
- `QRCodesPage.tsx` — sessionCode в QR URL
- `MobilePlayer.tsx` — sessionCode из URL params
- `RoleLobby.tsx` — sessionCode + навигация с параметром
- `MapProjection.tsx` — sessionCode из URL

### Архитектура
```
┌─────────────┐     ┌──────────────────────────────┐
│  Client A   │────▶│  WebSocket Room: session:ABC │
│  (Game 1)   │     │  State: PostgreSQL (ABC)     │
├─────────────┤     ├──────────────────────────────┤
│  Client B   │────▶│  WebSocket Room: session:XYZ │
│  (Game 2)   │     │  State: PostgreSQL (XYZ)     │
└─────────────┘     └──────────────────────────────┘
```

### Что проверить
1. Создать две игры от одного фасилитатора
2. Открыть админку Game A в одном браузере
3. Открыть админку Game B в другом браузере
4. Изменить состояние в Game A → Game B НЕ должен измениться
5. Запустить таймер в Game A → в Game B таймер НЕ запускается
6. Игроки Game A видят только своё состояние
7. Игроки Game B видят только своё состояние

### Breaking Changes
- ⚠️ SQLite больше не используется для игрового состояния
- ⚠️ Все API endpoints требуют параметр `?session=CODE`
- ⚠️ GameService методы теперь async (возвращают Promise)

## Фаза 1.2: Привязка игр к фасилитатору (завершено)

**Дата:** 2024-03-20

### Что сделано
- ✅ Создание игры через личный кабинет с выбором количества игроков
- ✅ Привязка сессии к facilitator_id в PostgreSQL
- ✅ Генерация уникальных кодов сессий (6 символов)
- ✅ Список игр фасилитатора с фильтрацией по статусу
- ✅ API для управления сессиями (`/api/sessions`)
- ✅ Проверка лимитов подписки (FREE: до 4 игроков)
- ✅ JWT авторизация в админке (без ввода пароля для залогиненных)
- ✅ Количество игроков сохраняется в сессии и используется везде
- ✅ Страница настройки: playerCount read-only, выбор сложности/режима
- ✅ Session code передаётся между страницами (`/admin` → `/setup` → `/qr`)
- ✅ Ссылка "Мои игры" в админке для возврата в кабинет

### Что проверить
1. Войти в `/facilitator`
2. Нажать "Создать игру" → выбрать количество игроков (4 для FREE)
3. Увидеть игру в списке со статусом "Подготовка"
4. Нажать "Управление" → попасть в админку автоматически (без пароля)
5. Нажать "Настройка игры" → увидеть количество игроков (read-only)
6. Выбрать сложность и режим распределения
7. Нажать "Начать распределение" → перейти к QR-кодам
8. Вернуться в личный кабинет по ссылке "Мои игры"

### Архитектура
```
/facilitator          → Создание игры (playerCount → PostgreSQL)
     ↓
/admin?session=CODE   → Админка (JWT авторизация)
     ↓
/setup?session=CODE   → Настройка (playerCount read-only, выбор сложности)
     ↓
/qr?session=CODE      → QR-коды для ролей
```

### Ограничения текущей версии
- ⚠️ Все игры используют общее игровое состояние (SQLite)
- ⚠️ Мультисессионность будет в Фазе 2
- ⚠️ Удалить можно только игры в статусе "Подготовка" или "Завершена"
