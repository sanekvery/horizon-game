# Фаза 1: Авторизация + PostgreSQL

**Статус:** ✅ Завершено
**Даты:** 2024-03-20

---

## Цель

Добавить авторизацию фасилитаторов и перейти на PostgreSQL для хранения данных.

---

## Выполнено

### Фаза 1.1: Авторизация фасилитаторов

- ✅ PostgreSQL добавлен в docker-compose
- ✅ Prisma ORM с миграциями
- ✅ Регистрация фасилитатора (email + пароль)
- ✅ Вход фасилитатора (JWT токены)
- ✅ Страница `/auth` — вход/регистрация
- ✅ Страница `/facilitator` — личный кабинет

### Фаза 1.2: Привязка игр к фасилитатору

- ✅ Создание игры через личный кабинет с выбором количества игроков
- ✅ Привязка сессии к `facilitator_id` в PostgreSQL
- ✅ Генерация уникальных кодов сессий (6 символов)
- ✅ Список игр фасилитатора с фильтрацией по статусу
- ✅ API для управления сессиями (`/api/sessions`)
- ✅ Проверка лимитов подписки (FREE: до 4 игроков)
- ✅ JWT авторизация в админке (без ввода пароля для залогиненных)
- ✅ Количество игроков сохраняется в сессии
- ✅ Session code передаётся между страницами

---

## Архитектура

```
/facilitator          → Создание игры (playerCount → PostgreSQL)
     ↓
/admin?session=CODE   → Админка (JWT авторизация)
     ↓
/setup?session=CODE   → Настройка (playerCount read-only)
     ↓
/qr?session=CODE      → QR-коды для ролей
```

- ⚠️ Мультисессионность будет в Фазе 2
