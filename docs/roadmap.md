## 2. Фазы разработки

### ✅ Фаза 1: Авторизация + PostgreSQL (ЗАВЕРШЕНО)

**Что сделано:**
- [x] Docker Compose с PostgreSQL
- [x] Prisma ORM + миграции
- [x] Регистрация/вход фасилитатора (JWT)
- [x] Личный кабинет фасилитатора
- [x] Создание игр с выбором количества игроков
- [x] Список своих игр с фильтрацией по статусу
- [x] Ограничение: >4 игроков требует подписку
- [x] JWT авторизация в админке
- [x] Session code передаётся между страницами
- [x] Количество игроков фиксируется при создании

**Результат:** Фасилитатор входит, создаёт игры, управляет ими

---

### 🔄 Фаза 2: Мультисессионность (СЛЕДУЮЩИЙ ШАГ)

**Цель:** Несколько игр одновременно, каждая изолирована

**Задачи:**
- [ ] Игровое состояние в PostgreSQL (вместо общего SQLite)
- [ ] Изоляция WebSocket комнат по session code
- [ ] QR-коды и токены привязаны к конкретной сессии
- [ ] Переключение между активными сессиями в ЛК
- [ ] Пауза/возобновление сессии

**Технические изменения:**
```
Сейчас:
  SQLite (один файл) → все сессии

После:
  PostgreSQL game_sessions.state → каждая сессия отдельно
  WebSocket rooms: session:{code} → изоляция событий
```

**Результат:** Фасилитатор может вести несколько игр одновременно
