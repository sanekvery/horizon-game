## 1. Бизнес-модель

### Freemium
```
┌─────────────────────────────────────────────────────────┐
│  БЕСПЛАТНО                                              │
│  ├── Игры до 4 человек               ✅ Реализовано     │

│  └── Без истории/аналитики                              │
├─────────────────────────────────────────────────────────┤
│  ПЛАТНО (разовая покупка или подписка)                  │
│  ├── Игры 5-20 человек               ✅ Ограничение     │
│  ├── История всех игр                                   │
│  ├── Аналитика и отчёты                                 │
│  ├── Система прогрессии персонажей                      │
│  ├── Дополнительные сценарии                            │
│  └── Приоритетная поддержка                             │
└─────────────────────────────────────────────────────────┘
```

### Пользователи системы

| Роль | Описание | Авторизация | Статус |
|------|----------|-------------|--------|
| **Гость** | Может играть по QR/ссылке, прогресс не сохраняется | Нет | ✅ Работает |
| **Игрок** | Играет с сохранением прогресса, личный кабинет | Email + пароль | ✅ Реализовано |
| **Фасилитатор** | Создаёт и ведёт игры, подписки | Логин + пароль | ✅ Реализовано |
| **Админ** | Управляет фасилитаторами, видит всё | Логин + пароль | 📋 Планируется |

## 4. Инфраструктура

### Деплой
После каждой фазы:
```bash
git pull origin master
docker-compose down
docker-compose up -d --build
docker-compose logs -f
```

### Переменные окружения
```
ADMIN_PASSWORD=...        # Пароль для legacy доступа к админке
DB_PASSWORD=...           # Пароль PostgreSQL
JWT_SECRET=...            # Секрет для JWT токенов
```
