## 1. Текущее состояние

### Что есть сейчас
```
┌─────────────────────────────────────────────────────────┐
│  SQLite (game.db)                                       │
│  └── Одна активная сессия                               │
│      └── JSON-blob с полным состоянием игры             │
├─────────────────────────────────────────────────────────┤
│  Server (Node.js + Socket.IO)                           │
│  └── Один процесс = одна игра                           │
├─────────────────────────────────────────────────────────┤
│  Client (React + Vite)                                  │
│  └── Статичный сценарий из JSON-файлов                  │
└─────────────────────────────────────────────────────────┘
```

### Ограничения
- ❌ Нет истории прошлых игр
- ❌ Нет логирования действий
- ❌ Одна сессия на сервер
- ❌ Сценарий не меняется от действий игроков
- ❌ Карта статична

## 3. Новые возможности

### 3.1 История игр и аналитика

**Что даёт:**
- Просмотр прошлых сессий
- Статистика по ролям и решениям
- Сравнение команд
- Экспорт отчётов для HR/тренеров

**Данные для сбора:**
```
┌─────────────────────────────────────────────────────────┐
│  СЕССИЯ                                                 │
│  ├── id, название, дата, продолжительность              │
│  ├── настройки (сложность, кол-во игроков)              │
│  ├── финальный результат (уровни зон, исход)            │
│  └── participants[]                                     │
│                                                         │
│  УЧАСТНИК                                               │
│  ├── имя, роль, время в игре                            │
│  ├── ресурсы: получено / потрачено                      │
│  ├── голосования: как голосовал                         │
│  └── обещание (если дал)                                │
│                                                         │
│  ЛОГ ДЕЙСТВИЙ                                           │
│  ├── timestamp, session_id, player_id                   │
│  ├── action_type (vote, contribute, reveal, etc.)       │
│  ├── action_data (JSON)                                 │
│  └── game_context (акт, сцена)                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 Логирование действий

**Типы событий для логирования:**

| Категория | События |
|-----------|---------|
| **Игрок** | вход, выход, выбор роли, голосование, вклад ресурсов, раскрытие секрета, обещание |
| **Фасилитатор** | смена сцены, запуск события, выдача ресурсов, улучшение зоны |
| **Система** | старт/конец игры, случайное событие, достижение порога |

**Формат лога:**
```json
{
  "id": "uuid",
  "session_id": "session-123",
  "timestamp": "2024-01-15T14:32:00Z",
  "actor": {
    "type": "player",
    "id": 5,
    "name": "Командор"
  },
  "action": "contribute",
  "data": {
    "zone": "center",
    "resource": "materials",
    "amount": 10
  },
  "context": {
    "act": 2,
    "scene": 5,
    "zone_level_before": 1,
    "zone_level_after": 1
  }
}
```

3. **Обучение** — преподаватель видит все сессии с одной панели

**Архитектура:**

## 4. Технический стек

### Текущий стек
- **Frontend:** React + TypeScript + Vite + Tailwind
- **Backend:** Node.js + Express + Socket.IO
- **Database:** SQLite (один файл)
- **State:** JSON в памяти + файл

### Предлагаемые изменения

| Компонент | Сейчас | Предложение | Почему |
|-----------|--------|-------------|--------|
| **Database** | SQLite | **PostgreSQL** | Параллельные сессии, сложные запросы, надёжность |
| **ORM** | Нет | **Prisma** | Type-safe, миграции, удобство |
| **Cache** | Нет | **Redis** | Быстрый доступ к состоянию сессий |
| **Sessions** | Один файл | **Отдельные записи в БД** | Изоляция, история |
| **Logs** | console.log | **Структурированные в БД** | Аналитика, replay |
| **Assets** | Статика | **S3/MinIO** | Кастомные карты, аватары |

### Схема базы данных (PostgreSQL)

```sql
-- Организации (для B2B)
CREATE TABLE organizations (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Игровые сессии
CREATE TABLE game_sessions (
  id UUID PRIMARY KEY,
  organization_id UUID REFERENCES organizations(id),
  code VARCHAR(8) UNIQUE, -- короткий код для входа
  name TEXT,
  status VARCHAR(20), -- 'setup', 'active', 'paused', 'finished'
  settings JSONB, -- сложность, кол-во игроков и т.д.
  state JSONB, -- текущее состояние игры
  started_at TIMESTAMP,
  finished_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Участники сессий
CREATE TABLE session_participants (
  id UUID PRIMARY KEY,
  session_id UUID REFERENCES game_sessions(id),
  role_id INT,
  player_name TEXT,
  token VARCHAR(16) UNIQUE,
  joined_at TIMESTAMP,
  left_at TIMESTAMP,
  is_connected BOOLEAN DEFAULT FALSE
);

-- Лог действий
CREATE TABLE action_logs (
  id UUID PRIMARY KEY,
  session_id UUID REFERENCES game_sessions(id),
  participant_id UUID REFERENCES session_participants(id),
  action_type VARCHAR(50),
  action_data JSONB,
  game_context JSONB, -- акт, сцена, состояние на момент действия
  created_at TIMESTAMP DEFAULT NOW()
);

-- События игры (случайные и триггерные)
CREATE TABLE game_events (
  id UUID PRIMARY KEY,
  session_id UUID REFERENCES game_sessions(id),
  event_type VARCHAR(50),
  event_id INT, -- ссылка на events.json
  triggered_by VARCHAR(50), -- 'random', 'facilitator', 'threshold'
  choice_made TEXT,
  effects_applied JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Индексы для быстрых запросов
CREATE INDEX idx_sessions_org ON game_sessions(organization_id);
CREATE INDEX idx_sessions_status ON game_sessions(status);
CREATE INDEX idx_logs_session ON action_logs(session_id);
CREATE INDEX idx_logs_time ON action_logs(created_at);
```
