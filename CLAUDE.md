# Инструкции для Claude Code

## Проект
**Проект «Горизонт»** — интерактивная ролевая игра для команд.

## Деплой

### ВАЖНО: После каждой завершённой фазы/фичи
1. Проверить сборку: `npm run build`
2. Закоммитить и запушить в git
3. Дать пользователю **точные инструкции** для обновления на сервере:

```bash
# Инструкции для сервера (копировать пользователю)
cd /path/to/horizon
git pull origin master
docker-compose down
docker-compose build
docker-compose up -d
docker-compose logs -f  # проверить логи
```

### Текущая инфраструктура
- **Хостинг:** Собственный сервер пользователя
- **Контейнеры:** Docker + Docker Compose
- **База данных:** PostgreSQL (в контейнере) — переход с SQLite
- **Кэш:** Redis (в контейнере) — будет добавлен

### Структура проекта
```
/client — React + Vite + TypeScript
/server — Node.js + Express + Socket.IO
/docs   — Документация
```

## Бизнес-контекст

### Монетизация
- **Бесплатно:** до 4 игроков
- **Платно:** 5-20 игроков, прогрессия персонажей, аналитика

### Пользователи
- **Гость** — играет по QR-коду (без авторизации)
- **Фасилитатор** — логин/пароль, личный кабинет, создаёт игры
- **Админ** — управляет всем

## Текущая фаза разработки

## Не забывать

1. После каждого изменения — проверить `npm run build`
2. Всегда пушить в git перед инструкциями по деплою
3. Docker-first подход — минимум настройки на сервере
4. Пользователь не хочет "танцы с бубном" — давать копируемые команды

## Документирование работ

**ВАЖНО:** После каждого этапа работ (фичи, дебага, доработки):

1. **Создать документ в `docs/phases/`:**
   - Формат: `phase-X.X-название.md` или `phase-X.X-название-fix.md`
   - Содержание: проблема → решение → что изменено → результат

2. **Обновить `docs/roadmap.md`:**
   - Добавить ссылку на новый документ
   - Отметить что сделано с датой
   - Обновить дату документа

3. **Структура документа работ:**
   ```markdown
   # Фаза X.X: Название — [Доработка/Исправление/...]

   **Дата:** YYYY-MM-DD
   **Статус:** Завершено/В работе

   ## Проблема
   Что было не так, почему потребовалась работа

   ## Решение
   Что сделано, какие файлы изменены

   ## Результат
   Что работает теперь, как проверить
   ```

Это нужно чтобы видеть историю: от чего шли → к чему пришли.
